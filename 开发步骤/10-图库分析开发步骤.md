# 图库分析 - 开发步骤指南

本文档基于"智能协同云图库项目教程"第10节整理，涵盖空间资源使用分析、分类分析、标签分析、大小分析、用户上传行为分析、空间排行分析六大功能模块的完整开发步骤。

---

## 一、需求分析

### 用户空间图库分析

为空间（私有 / 团队）所有者提供空间维度下的分析能力：

1. **空间资源使用分析**：查看已使用存储空间和图片数量，以及最大限制和百分比
2. **空间图片分类分析**：按分类统计图片数量和总大小
3. **空间图片标签分析**：统计各标签的使用频率
4. **空间图片大小分析**：按大小范围（< 100KB、100KB-500KB、500KB-1MB、> 1MB）统计分布
5. **用户上传行为分析**：按时间维度（日 / 周 / 月）统计上传趋势

### 管理员全空间分析

管理员可跨空间查看所有的空间分析，额外支持：

6. **空间使用排行分析**：按存储空间使用量排名，查看 Top N 个空间

---

## 二、方案设计

### 1、分析类需求的实现流程

```
用户选择分析范围 → 后端接收请求 → 拼装 SQL 查询 + 分组统计 → 聚合结果 → 返回数据 → 前端可视化渲染
```

### 2、本项目实现方案

- **后端**：新建 `SpaceAnalyzeController` + `SpaceAnalyzeService`，统一管理分析接口
- **分析范围**：指定空间 / 全空间 / 公共图库三种范围
- **前端**：使用 ECharts 进行数据可视化，每个分析维度封装为独立组件，在统一分析页面组合展示

## 三、后端开发

### 通用分析请求基类

新建通用分析请求父类 `SpaceAnalyzeRequest`，提取公共属性（查询范围），其他分析请求类继承它：

```java
@Data
public class SpaceAnalyzeRequest implements Serializable {

    /**
     * 空间 ID（指定空间分析时使用）
     */
    private Long spaceId;

    /**
     * 是否查询所有空间（管理员权限）
     */
    private boolean queryAll;

    /**
     * 是否查询公共图库
     */
    private boolean queryPublic;

    private static final long serialVersionUID = 1L;
}
```

**通用查询条件填充方法**（放在 SpaceAnalyzeService 中）：

```java
/**
 * 根据分析范围补充查询条件
 */
private void fillAnalyzeQueryWrapper(SpaceAnalyzeRequest spaceAnalyzeRequest, QueryWrapper<Picture> queryWrapper) {
    if (spaceAnalyzeRequest.isQueryAll()) {
        // 查询全部空间，不需要过滤
        return;
    }
    if (spaceAnalyzeRequest.isQueryPublic()) {
        // 查询公共图库（spaceId 为 null）
        queryWrapper.isNull("spaceId");
        return;
    }
    Long spaceId = spaceAnalyzeRequest.getSpaceId();
    if (spaceId != null) {
        queryWrapper.eq("spaceId", spaceId);
    } else {
        throw new BusinessException(ErrorCode.PARAMS_ERROR, "未指定分析范围");
    }
}
```

**权限校验方法**（处理用户和管理员的权限逻辑）：

```java
/**
 * 校验空间分析权限
 */
private void checkSpaceAnalyzeAuth(SpaceAnalyzeRequest spaceAnalyzeRequest, User loginUser) {
    if (spaceAnalyzeRequest.isQueryAll() || spaceAnalyzeRequest.isQueryPublic()) {
        // 查询全部/公共图库，仅管理员可以访问
        ThrowUtils.throwIf(!userService.isAdmin(loginUser), ErrorCode.NO_AUTH_ERROR);
    } else {
        // 查询指定空间，校验空间权限
        Long spaceId = spaceAnalyzeRequest.getSpaceId();
        Space space = spaceService.getById(spaceId);
        ThrowUtils.throwIf(space == null, ErrorCode.NOT_FOUND_ERROR, "空间不存在");
        spaceService.checkSpaceAuth(loginUser, space);
    }
}
```

**空间权限校验方法**（封装到 SpaceService 中，供多处复用）：

```java
/**
 * 空间权限校验（仅本人或管理员可访问）
 */
@Override
public void checkSpaceAuth(User loginUser, Space space) {
    if (!space.getUserId().equals(loginUser.getId()) && !userService.isAdmin(loginUser)) {
        throw new BusinessException(ErrorCode.NO_AUTH_ERROR);
    }
}
```

> 提示：将 SpaceController 中编辑和删除操作的权限校验代码也替换为 `checkSpaceAuth` 方法，统一权限逻辑。

#### 1、空间资源使用分析

**请求类 `SpaceUsageAnalyzeRequest`：**

```java
@EqualsAndHashCode(callSuper = true)
@Data
public class SpaceUsageAnalyzeRequest extends SpaceAnalyzeRequest {
}
```

**响应类 `SpaceUsageAnalyzeResponse`：**

```java
@Data
public class SpaceUsageAnalyzeResponse implements Serializable {

    /** 已使用大小 */
    private Long usedSize;
    /** 最大大小限制 */
    private Long maxSize;
    /** 空间使用比例 */
    private Double sizeUsageRatio;
    /** 当前图片数量 */
    private Long usedCount;
    /** 最大图片数量限制 */
    private Long maxCount;
    /** 图片数量使用比例 */
    private Double countUsageRatio;

    private static final long serialVersionUID = 1L;
}
```

**Service 实现：**

```java
@Override
public SpaceUsageAnalyzeResponse getSpaceUsageAnalyze(SpaceUsageAnalyzeRequest spaceUsageAnalyzeRequest, User loginUser) {
    ThrowUtils.throwIf(spaceUsageAnalyzeRequest == null, ErrorCode.PARAMS_ERROR);
    if (spaceUsageAnalyzeRequest.isQueryAll() || spaceUsageAnalyzeRequest.isQueryPublic()) {
        // 查询全部或公共图库逻辑，仅管理员可访问
        boolean isAdmin = userService.isAdmin(loginUser);
        ThrowUtils.throwIf(!isAdmin, ErrorCode.NO_AUTH_ERROR, "无权访问空间");
        // 统计公共图库的资源使用（只查 picSize 列，提高性能）
        QueryWrapper<Picture> queryWrapper = new QueryWrapper<>();
        queryWrapper.select("picSize");
        if (!spaceUsageAnalyzeRequest.isQueryAll()) {
            queryWrapper.isNull("spaceId");
        }
        List<Object> pictureObjList = pictureService.getBaseMapper().selectObjs(queryWrapper);
        long usedSize = pictureObjList.stream().mapToLong(result -> result instanceof Long ? (Long) result : 0).sum();
        long usedCount = pictureObjList.size();
        // 封装返回结果
        SpaceUsageAnalyzeResponse spaceUsageAnalyzeResponse = new SpaceUsageAnalyzeResponse();
        spaceUsageAnalyzeResponse.setUsedSize(usedSize);
        spaceUsageAnalyzeResponse.setUsedCount(usedCount);
        // 公共图库无上限、无比例
        spaceUsageAnalyzeResponse.setMaxSize(null);
        spaceUsageAnalyzeResponse.setSizeUsageRatio(null);
        spaceUsageAnalyzeResponse.setMaxCount(null);
        spaceUsageAnalyzeResponse.setCountUsageRatio(null);
        return spaceUsageAnalyzeResponse;
    } else {
        // 查询指定空间
        Long spaceId = spaceUsageAnalyzeRequest.getSpaceId();
        ThrowUtils.throwIf(spaceId == null || spaceId <= 0, ErrorCode.PARAMS_ERROR);
        Space space = spaceService.getById(spaceId);
        ThrowUtils.throwIf(space == null, ErrorCode.NOT_FOUND_ERROR, "空间不存在");
        spaceService.checkSpaceAuth(loginUser, space);

        SpaceUsageAnalyzeResponse response = new SpaceUsageAnalyzeResponse();
        response.setUsedSize(space.getTotalSize());
        response.setMaxSize(space.getMaxSize());
        // 后端算好百分比，前端直接展示
        double sizeUsageRatio = NumberUtil.round(space.getTotalSize() * 100.0 / space.getMaxSize(), 2).doubleValue();
        response.setSizeUsageRatio(sizeUsageRatio);
        response.setUsedCount(space.getTotalCount());
        response.setMaxCount(space.getMaxCount());
        double countUsageRatio = NumberUtil.round(space.getTotalCount() * 100.0 / space.getMaxCount(), 2).doubleValue();
        response.setCountUsageRatio(countUsageRatio);
        return response;
    }
}
```

> 优化细节：查询公共图库时，使用 `queryWrapper.select("picSize")` 只查询需要的列，并通过 `selectObjs` 直接返回 Object 对象，避免封装为 Picture 对象，提高性能。

**Controller 接口：**

```java

    @Resource
    private SpaceAnalyzeService spaceAnalyzeService;
    @Resource
    private UserService userService;
```

```java
@Override
public List<SpaceCategoryAnalyzeResponse> getSpaceCategoryAnalyze(SpaceCategoryAnalyzeRequest spaceCategoryAnalyzeRequest, User loginUser) {
    ThrowUtils.throwIf(spaceCategoryAnalyzeRequest == null, ErrorCode.PARAMS_ERROR);
    checkSpaceAnalyzeAuth(spaceCategoryAnalyzeRequest, loginUser);

    QueryWrapper<Picture> queryWrapper = new QueryWrapper<>();
    fillAnalyzeQueryWrapper(spaceCategoryAnalyzeRequest, queryWrapper);

    // 按分类分组统计
    queryWrapper.select("category AS category",
                    "COUNT(*) AS count",
                    "SUM(picSize) AS totalSize")
            .groupBy("category");

    return pictureService.getBaseMapper().selectMaps(queryWrapper)
            .stream()
            .map(result -> {
                String category = result.get("category") != null ? result.get("category").toString() : "未分类";
                Long count = ((Number) result.get("count")).longValue();
                Long totalSize = ((Number) result.get("totalSize")).longValue();
                return new SpaceCategoryAnalyzeResponse(category, count, totalSize);
            })
            .collect(Collectors.toList());
}
```

#### 3、空间图片标签分析

由于图片的标签字段（tags）是 JSON 数组存储（如 `["风景","自然"]`），需要先查出所有标签字符串，再在 Java 内存中合并和统计。

**请求类 `SpaceTagAnalyzeRequest`：**

```java
@EqualsAndHashCode(callSuper = true)
@Data
public class SpaceTagAnalyzeRequest extends SpaceAnalyzeRequest {
}
```

**响应类 `SpaceTagAnalyzeResponse`：**

```java
@Data
@AllArgsConstructor
@NoArgsConstructor
public class SpaceTagAnalyzeResponse implements Serializable {
    /** 标签名称 */
    private String tag;
    /** 使用次数 */
    private Long count;
    private static final long serialVersionUID = 1L;
}
```

**Service 实现：**

```java
@Override
public List<SpaceTagAnalyzeResponse> getSpaceTagAnalyze(SpaceTagAnalyzeRequest spaceTagAnalyzeRequest, User loginUser) {
    ThrowUtils.throwIf(spaceTagAnalyzeRequest == null, ErrorCode.PARAMS_ERROR);
    checkSpaceAnalyzeAuth(spaceTagAnalyzeRequest, loginUser);

    QueryWrapper<Picture> queryWrapper = new QueryWrapper<>();
    fillAnalyzeQueryWrapper(spaceTagAnalyzeRequest, queryWrapper);

    // 只查询 tags 字段
    queryWrapper.select("tags");
    List<String> tagsJsonList = pictureService.getBaseMapper().selectObjs(queryWrapper)
            .stream()
            .filter(ObjUtil::isNotNull)
            .map(Object::toString)
            .collect(Collectors.toList());

    // 合并所有标签并统计使用次数
    Map<String, Long> tagCountMap = tagsJsonList.stream()
            .flatMap(tagsJson -> JSONUtil.toList(tagsJson, String.class).stream())
            .collect(Collectors.groupingBy(tag -> tag, Collectors.counting()));

    // 转换为响应对象，按使用次数降序排序
    return tagCountMap.entrySet().stream()
            .sorted((e1, e2) -> Long.compare(e2.getValue(), e1.getValue()))
            .map(entry -> new SpaceTagAnalyzeResponse(entry.getKey(), entry.getValue()))
            .collect(Collectors.toList());
}
```

**Controller 接口：**

```java
@PostMapping("/tag")
public BaseResponse<List<SpaceTagAnalyzeResponse>> getSpaceTagAnalyze(
        @RequestBody SpaceTagAnalyzeRequest spaceTagAnalyzeRequest,
        HttpServletRequest request) {
    ThrowUtils.throwIf(spaceTagAnalyzeRequest == null, ErrorCode.PARAMS_ERROR);
    User loginUser = userService.getLoginUser(request);
    List<SpaceTagAnalyzeResponse> resultList = spaceAnalyzeService.getSpaceTagAnalyze(spaceTagAnalyzeRequest, loginUser);
```

**请求类 `SpaceSizeAnalyzeRequest`：**

**响应类 `SpaceSizeAnalyzeResponse`：**

```java
@NoArgsConstructor

    private String sizeRange;
    /** 图片数量 */
    private Long count;
    private static final long serialVersionUID = 1L;
}
```

**Service 实现**（分段统计）：

```java
@Override
public List<SpaceSizeAnalyzeResponse> getSpaceSizeAnalyze(SpaceSizeAnalyzeRequest spaceSizeAnalyzeRequest, User loginUser) {
    ThrowUtils.throwIf(spaceSizeAnalyzeRequest == null, ErrorCode.PARAMS_ERROR);
    checkSpaceAnalyzeAuth(spaceSizeAnalyzeRequest, loginUser);

    QueryWrapper<Picture> queryWrapper = new QueryWrapper<>();
    fillAnalyzeQueryWrapper(spaceSizeAnalyzeRequest, queryWrapper);

    // 只查询 picSize 字段
    queryWrapper.select("picSize");
    List<Long> picSizes = pictureService.getBaseMapper().selectObjs(queryWrapper)
            .stream()
            .map(size -> ((Number) size).longValue())
            .collect(Collectors.toList());

    // 使用 LinkedHashMap 保持分段顺序
    Map<String, Long> sizeRanges = new LinkedHashMap<>();
    sizeRanges.put("<100KB", picSizes.stream().filter(size -> size < 100 * 1024).count());
    sizeRanges.put("100KB-500KB", picSizes.stream().filter(size -> size >= 100 * 1024 && size < 500 * 1024).count());
    sizeRanges.put("500KB-1MB", picSizes.stream().filter(size -> size >= 500 * 1024 && size < 1 * 1024 * 1024).count());
    sizeRanges.put(">1MB", picSizes.stream().filter(size -> size >= 1 * 1024 * 1024).count());

    return sizeRanges.entrySet().stream()
            .map(entry -> new SpaceSizeAnalyzeResponse(entry.getKey(), entry.getValue()))
            .collect(Collectors.toList());
}
```

> 优化：上面的代码遍历了 4 次 picSizes，可以优化为只遍历一次同时分段统计。

#### 5、用户上传行为分析

支持按时间维度（日 / 周 / 月）统计，使用 MySQL 日期函数 `DATE_FORMAT` 和 `YEARWEEK` 进行分组。

**请求类 `SpaceUserAnalyzeRequest`：**

```java
@EqualsAndHashCode(callSuper = true)
@Data
public class SpaceUserAnalyzeRequest extends SpaceAnalyzeRequest {
    /** 用户 ID */
    private Long userId;
    /** 时间维度：day / week / month */
    private String timeDimension;
}
```

**响应类 `SpaceUserAnalyzeResponse`：**

```java
@Data
@AllArgsConstructor
@NoArgsConstructor
public class SpaceUserAnalyzeResponse implements Serializable {
    /** 时间区间 */
    private String period;
    /** 上传数量 */
    private Long count;
    private static final long serialVersionUID = 1L;
}
```

**Service 实现：**

```java
@Override
public List<SpaceUserAnalyzeResponse> getSpaceUserAnalyze(SpaceUserAnalyzeRequest spaceUserAnalyzeRequest, User loginUser) {
    ThrowUtils.throwIf(spaceUserAnalyzeRequest == null, ErrorCode.PARAMS_ERROR);
    checkSpaceAnalyzeAuth(spaceUserAnalyzeRequest, loginUser);

    QueryWrapper<Picture> queryWrapper = new QueryWrapper<>();
    Long userId = spaceUserAnalyzeRequest.getUserId();
    queryWrapper.eq(ObjUtil.isNotNull(userId), "userId", userId);
    fillAnalyzeQueryWrapper(spaceUserAnalyzeRequest, queryWrapper);

    // 根据时间维度选择不同的 MySQL 函数
    String timeDimension = spaceUserAnalyzeRequest.getTimeDimension();
    switch (timeDimension) {
        case "day":
            queryWrapper.select("DATE_FORMAT(createTime, '%Y-%m-%d') AS period", "COUNT(*) AS count");
            break;
        case "week":
            queryWrapper.select("YEARWEEK(createTime) AS period", "COUNT(*) AS count");
            break;
        case "month":
            queryWrapper.select("DATE_FORMAT(createTime, '%Y-%m') AS period", "COUNT(*) AS count");
            break;
        default:
            throw new BusinessException(ErrorCode.PARAMS_ERROR, "不支持的时间维度");
    }

    // 分组和排序
    queryWrapper.groupBy("period").orderByAsc("period");

    // 查询结果并转换
    List<Map<String, Object>> queryResult = pictureService.getBaseMapper().selectMaps(queryWrapper);
    return queryResult.stream()
            .map(result -> {
                String period = result.get("period").toString();
                Long count = ((Number) result.get("count")).longValue();
                return new SpaceUserAnalyzeResponse(period, count);
            })
            .collect(Collectors.toList());
}
```

**Controller 接口：**

```java
@PostMapping("/user")

        @RequestBody SpaceUserAnalyzeRequest spaceUserAnalyzeRequest,
        HttpServletRequest request) {
    ThrowUtils.throwIf(spaceUserAnalyzeRequest == null, ErrorCode.PARAMS_ERROR);
    User loginUser = userService.getLoginUser(request);
    List<SpaceUserAnalyzeResponse> resultList = spaceAnalyzeService.getSpaceUserAnalyze(spaceUserAnalyzeRequest, loginUser);
```

### 扩展知识 - 自定义 SQL

当 MyBatis-Plus 提供的方法无法满足复杂查询需求时，可通过两种方式自定义 SQL。

#### 1、Java 注解实现

在 Mapper 接口中直接定义方法，用 `@Select` 注解写 SQL：

```java
public interface PictureMapper extends BaseMapper<Picture> {
    
    @Select("SELECT category, COUNT(*) as count, SUM(picSize) as totalSize " +
            "FROM picture WHERE space_id = #{spaceId} GROUP BY category")
    List<SpaceCategoryAnalyzeResponse> analyzeCategory(Long spaceId);
}
```

#### 2、XML 配置实现

在 `resources/mapper/PictureMapper.xml` 中编写更复杂的 SQL：

```xml
<select id="analyzeCategory" resultType="com.xxx.model.vo.SpaceCategoryAnalyzeResponse">
    SELECT category, COUNT(*) as count, SUM(picSize) as totalSize
    FROM picture
    WHERE space_id = #{spaceId}
    GROUP BY category
</select>
```

### 扩展知识 - 查询加速

对于大数据量分析场景，可以考虑：
- 对 `spaceId`、`userId`、`createTime` 等高频查询字段建立索引
- 使用缓存存储分析结果（如每小时刷新一次）
- 对于超大规模数据，考虑异步任务 + 定时汇总

```typescript
const fetchData = async () => {
```

## 关键技术点

| 技术点 | 说明 |
|--------|------|
| **MyBatis-Plus 分组查询** | `queryWrapper.select().groupBy()` + `selectMaps()` 返回 Map 列表 |
| **selectObjs 优化** | 只查需要的列，通过 `selectObjs` 返回原始类型，避免封装对象 |
| **MySQL 日期函数** | `DATE_FORMAT(createTime, '%Y-%m-%d')`、`YEARWEEK(createTime)` 实现时间维度分组 |
| **Java Stream 统计** | `groupingBy` + `counting` / `sorted` + `limit` 进行内存聚合 |
| **LinkedHashMap** | 保持分段顺序（普通 HashMap 无序） |
| **双 Y 轴图表** | ECharts 的 `yAxisIndex` 配置实现一个图表展示两种维度 |
| **watchEffect** | Vue 3 的自动依赖追踪，Props 变化时自动重新加载数据 |
| **computed 图表选项** | 使用 computed 根据 dataList 动态计算图表配置 |
| **前后端权限双重控制** | 前端 v-if 控制 UI 可见性 + 后端校验作为安全底线 |
| **公共父类继承** | SpaceAnalyzeRequest 作为请求基类，各子类继承复用查询范围参数 |
