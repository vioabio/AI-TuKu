# 部署上线 - 开发步骤指南

本文档基于"智能协同云图库项目教程"第14节整理，涵盖从服务器初始化到前后端部署上线的完整流程，包括宝塔面板操作、Nginx 配置、WebSocket 代理、Docker Compose 等。

---

## 一、服务器初始化

### 选择服务器

推荐使用**轻量应用服务器**（2核2G即可），预装宝塔 Linux 面板模板。

### 初始配置步骤

1. **开放端口**：在腾讯云防火墙中放通 `8888`（宝塔面板端口）
2. **获取宝塔账号密码**：通过服务器 Web 终端执行命令获取
3. **登录宝塔面板**：访问 `http://服务器IP:8888`
4. **安装环境**：首次进入推荐安装 **LNMP**（包含 Nginx），适合前后端分离项目
5. **修改面板密码**：首次登录后务必修改默认账号密码

## 二、部署规划

### 获取源码

本项目代码开源：[GitHub - yu-picture](https://github.com/liyupi/yu-picture)

| 版本 | 目录 | 适用人群 |
|------|------|----------|
| 传统分层架构 | `yu-picture-backend` | 新手 |
| DDD 架构 | `yu-picture-backend-ddd` | 有经验同学 |

> 两种架构的部署方式一致。

### 部署方案

| 端 | 部署方式 |
|----|----------|
| **后端** | 宝塔 Java 项目管理器运行 jar 包 |
| **前端** | Nginx 静态文件托管 |

### 地址规划

| 服务 | 访问地址 | 实际端口 |
|------|----------|----------|
| 前端 | `http://{域名}/` | 80 (Nginx) |
| 后端 | `http://{域名}/api` | 8123 (Java) |
| 数据库 | - | 3306 |
| Redis | - | 6379 |

> 使用 Nginx 转发使前后端域名一致，**避免跨域问题**。

### 注意事项

- 在腾讯云防火墙中放通 3306（MySQL）和 6379（Redis）端口
- JDK 建议选择 **17 版本**

## 三、安装依赖

### 1、数据库 MySQL

宝塔面板已预装 MySQL，直接使用即可：

1. 创建数据库，名称与项目配置一致（`yu_picture`）
2. 设置用户名和密码
3. 在 IDEA 中测试本地能否连接远程数据库
4. 执行 SQL 脚本初始化库表

### 2、Redis

1. 宝塔软件商店搜索安装 **Redis**
2. 配置 Redis：
   - 开启**远程访问**
   - 设置**密码**
3. **重载配置**使修改生效
4. 在 IDEA 中验证本地能否连接远程 Redis

### 3、Java 环境

在宝塔面板中快速安装 **JDK 17**（建议同时安装 JDK 8、11、17 多版本便于切换）。

### 4、其他服务

| 服务 | 操作 |
|------|------|
| 腾讯云 COS 对象存储 | 到官网开通，**需要配置服务器 IP 或前端域名的跨域规则** |
| 阿里云百炼 AI | 到官网开通 |

## 四、后端部署

### 1、修改配置

修改 `application-prod.yml` 生产环境配置：

```yaml
# 线上配置文件
server:
  port: 8123
spring:
  # 数据库配置（需替换为实际值）
  datasource:
    driver-class-name: com.mysql.cj.jdbc.Driver
    url: jdbc:mysql://{服务器IP}:3306/yu_picture
    username: {数据库用户名}
    password: {数据库密码}
  # Redis 配置（需替换为实际值）
  redis:
    database: 0
    host: {服务器IP}
    port: 6379
    timeout: 5000
    password: {Redis密码}
mybatis-plus:
  configuration:
    # 生产环境关闭日志
    log-impl: ''
# 接口文档配置（生产环境建议加密码保护）
knife4j:
  basic:
    enable: true
    username: root
    password: 123456
# 对象存储配置（需替换为实际值）
cos:
  client:
    host: xxx
    secretId: xxx
    secretKey: xxx
    region: xxx
    bucket: xxx
# 阿里云 AI 配置（需替换为实际值）
aliYunAi:
  apiKey: xxx
```

> 关键修改：数据库地址/密码、Redis 地址/密码、关闭 MyBatis Plus SQL 日志、Knife4j 加密码保护、COS 和 AI 的 key。

### 2、打包部署

**pom.xml 打包配置**：

```xml
<build>
    <plugins>
        <plugin>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-maven-plugin</artifactId>
            <version>${spring-boot.version}</version>
            <configuration>
                <mainClass>com.yupi.yupicturebackend.YuPictureBackendApplication</mainClass>
                <skip>true</skip>
            </configuration>
        </plugin>
    </plugins>
</build>
```

**部署步骤**：

1. 执行 `mvn package` 打包为 jar
2. 将 jar 包上传到服务器
3. 在宝塔 **Java 项目管理器** 中添加项目
4. 选择 JDK 版本（17），配置启动参数 `--spring.profiles.active=prod`
5. 启动项目，查看日志确认运行正常

### 3、Nginx 转发

**后端 API 代理**：

```nginx
location /api {
  proxy_pass  http://127.0.0.1:8123;
  proxy_set_header Host $proxy_host;
  proxy_set_header X-Real-IP $remote_addr;
  proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
  proxy_buffering off;
  proxy_set_header Connection "";
}
```

**WebSocket 代理**（协作编辑功能需要，注意是 `/api/ws`）：

```nginx
# 代理 WebSocket 连接 (专门用于 WebSocket 请求)
location /api/ws {
  proxy_pass http://127.0.0.1:8123;
  proxy_http_version 1.1;
  proxy_set_header Upgrade $http_upgrade;
  proxy_set_header Connection "upgrade";
  proxy_set_header Host $host;
  proxy_set_header X-Real-IP $remote_addr;
  proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
  proxy_buffering off;
  proxy_read_timeout 86400s;
}
```

## 五、前端部署

### 1、修改配置

**修改 Axios 请求地址**（`request.ts`）：

```typescript
// 区分开发和生产环境
const DEV_BASE_URL = "http://localhost:8123";
const PROD_BASE_URL = "http://{服务器IP}";
// 创建 Axios 实例
const myAxios = axios.create({
  baseURL: PROD_BASE_URL,
  timeout: 10000,
  withCredentials: true,
});
```

**修改 WebSocket 连接地址**（`pictureEditWebSocket.ts`）：

```typescript
const DEV_BASE_URL = "ws://localhost:8123";
const PROD_BASE_URL = "ws://{服务器IP}";
const url = `${PROD_BASE_URL}/api/ws/picture/edit?pictureId=${this.pictureId}`
```

### 2、打包部署

**定义 pure-build 命令**（`package.json`）：

```json
{
  "scripts": {
    "dev": "vite",
    "pure-build": "vite build",
    "build": "run-p type-check \"build-only {@}\" --"
  }
}
```

> 为什么用 `pure-build`？脚手架内置的 `build` 会执行类型检查，如有不规范会导致打包失败。`pure-build` 跳过类型检查，直接构建。

**部署步骤**：

1. 执行 `npm run pure-build` 构建
2. 构建成功后得到 `dist` 目录
3. 将 `dist` 下所有文件上传到服务器（建议先压缩再上传再解压）
4. 注意 Node.js 版本过低会导致构建失败，建议使用 **v20.17.0** 等 LTS 版本

### 3、Nginx 转发

修改站点配置，将网站目录指向前端文件根目录。

**Vue SPA 路由兼容配置**（解决刷新404问题）：

```nginx
location / {
  try_files $uri $uri/index.html /index.html;
}
```

> Vue 是单页面应用，打包后只有 `index.html`。直接访问子路由（如 `/add_picture`）时服务器上不存在对应文件，需要配置 Nginx fallback 到 `index.html`。

## 六、测试验证

部署完成后进行全面测试：

1. 访问前端首页是否正常加载
2. 用户注册/登录功能是否正常
3. 图片上传功能是否正常（COS 跨域是否配置正确）
4. 图片编辑/查看功能是否正常
5. 团队空间功能是否正常
6. WebSocket 协作编辑是否正常工作
7. 刷新子页面不会出现 404
8. 所有需要登录的操作在未登录时是否正确拦截

## 七、扩展知识 - Docker Compose 部署

如果不想用宝塔面板，也可以使用 Docker + Docker Compose 快速部署：

```yaml
# Docker Compose 文件，用于 Spring Boot 项目，依赖 MySQL 和 Redis
version: '3.8'

services:
  # MySQL 数据库服务
  mysql:
    image: mysql:8.0
    container_name: mysql_db
    restart: always
    environment:
      MYSQL_ROOT_PASSWORD: root
      MYSQL_DATABASE: yu_picture
      MYSQL_USER: root
      MYSQL_PASSWORD: 123456
      TZ: Asia/Shanghai
    ports:
      - "3306:3306"
    volumes:
      - mysql_data:/var/lib/mysql
    command: --default-authentication-plugin=mysql_native_password

  # Redis 缓存服务
  redis:
    image: redis:5.0
    container_name: redis_cache
    restart: always
    ports:
      - "6379:6379"
    volumes:
      - redis_data:/data
    environment:
      TZ: Asia/Shanghai

  # Spring Boot 应用服务
  springboot_app:
    image: openjdk:11-jre-slim
    container_name: springboot_app
    working_dir: /app
    volumes:
      - .:/app
    ports:
      - "8123:8123"
    environment:
      TZ: Asia/Shanghai
    command:
      [ "java", "-jar", "target/yu-picture-backend-0.0.1-SNAPSHOT.jar",
        "--spring.profiles.active=prod" ]
    depends_on:
      - mysql
      - redis

# 使用 Docker 管理的命名卷
volumes:
  mysql_data:
  redis_data:
```

## 九、核心 Nginx 配置汇总

将以下配置整合到 Nginx 站点配置中：

```nginx
server {
    listen       80;
    server_name  {你的域名或IP};

    # 前端静态文件
    location / {
        root   /www/wwwroot/yu-picture-frontend/dist;
        index  index.html index.htm;
        try_files $uri $uri/index.html /index.html;
    }

    # 后端 API 代理
    location /api {
        proxy_pass  http://127.0.0.1:8123;
        proxy_set_header Host $proxy_host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_buffering off;
        proxy_set_header Connection "";
    }

    # WebSocket 代理
    location /api/ws {
        proxy_pass http://127.0.0.1:8123;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_buffering off;
        proxy_read_timeout 86400s;
    }
}
```

## 关键注意事项

| 注意点 | 说明 |
|--------|------|
| **生产环境关闭 SQL 日志** | `mybatis-plus.configuration.log-impl: ''` |
| **接口文档加密码** | Knife4j 生产环境必须加 basic auth |
| **Redis 必须设密码** | 未设密码会被攻击 |
| **COS 配置跨域规则** | 添加服务器 IP 或前端域名到 COS 跨域白名单 |
| **前端路由 fallback** | `try_files $uri /index.html` 解决 SPA 刷新 404 |
| **WebSocket 单独代理** | `/api/ws` 需特殊配置 `Upgrade` 和 `Connection` 头 |
| **pure-build 跳过类型检查** | 避免因 TypeScript 类型问题打包失败 |
| **JDK 17** | 建议使用 JDK 17 版本 |
