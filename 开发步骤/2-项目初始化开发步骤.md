# 2 - 项目初始化 - 开发步骤指南

本文档基于"智能协同云图库项目教程"第2节整理，涵盖后端和前端两个项目的从零搭建，包括环境准备、依赖整合、通用基础代码、工程化配置、路由、请求封装等全套初始化流程。

---

## 一、后端项目初始化

### 环境准备

| 依赖 | 版本要求 | 说明 |
|------|----------|------|
| **JDK** | 8、11 或 17，推荐 11 | 不能超过 17！Caffeine 要求 JDK 11 |
| **MySQL** | 8.x 或 5.7 | 建议 8.x 版本 |
| **Maven** | 3.6+ | 包管理器 |

### 新建项目

1. IDEA 新建项目，选择 **Spring Initializr** 模板
2. 替换 Server URL 为 `https://start.aliyun.com/`（官方 URL 不支持选择 Java 8）
3. 选择 **Spring Boot 2.7.6** 版本
4. 添加依赖：Spring Web、MyBatis、MySQL Driver、Lombok

**基础配置文件 `application.yml`：**

```yaml
server:
  port: 8123
  servlet:
    context-path: /api
spring:
  application:
    name: yu-picture-backend
  datasource:
    driver-class-name: com.mysql.cj.jdbc.Driver
    url: jdbc:mysql://localhost:3306/yu_picture
    username: root
    password: root
```

#### 1、自定义异常 - ErrorCode

```java
public enum ErrorCode {

    SUCCESS(0, "ok"),
    PARAMS_ERROR(40000, "请求参数错误"),
    NOT_LOGIN_ERROR(40100, "未登录"),
    NO_AUTH_ERROR(40101, "无权限"),
    NOT_FOUND_ERROR(40400, "请求数据不存在"),
    FORBIDDEN_ERROR(40300, "禁止访问"),
    SYSTEM_ERROR(50000, "系统内部异常"),
    OPERATION_ERROR(50001, "操作失败");

    private final int code;
    private final String message;

    ErrorCode(int code, String message) {
        this.code = code;
        this.message = message;
    }

    public int getCode() { return code; }
    public String getMessage() { return message; }
}
```

#### 2、自定义异常 - BusinessException

```java
public class BusinessException extends RuntimeException {

    private final int code;

    public BusinessException(int code, String message) {
        super(message);
        this.code = code;
    }

    public BusinessException(ErrorCode errorCode) {
        super(errorCode.getMessage());
        this.code = errorCode.getCode();
    }

    public BusinessException(ErrorCode errorCode, String message) {
        super(message);
        this.code = errorCode.getCode();
    }

    public int getCode() { return code; }
}
```

#### 3、异常抛出工具 - ThrowUtils

```java
public class ThrowUtils {

    /**
     * 条件成立则抛异常
     */
    public static void throwIf(boolean condition, RuntimeException runtimeException) {
        if (condition) {
            throw runtimeException;
        }
    }

    public static void throwIf(boolean condition, ErrorCode errorCode) {
        throwIf(condition, new BusinessException(errorCode));
    }

    public static void throwIf(boolean condition, ErrorCode errorCode, String message) {
        throwIf(condition, new BusinessException(errorCode, message));
    }
}
```

#### 4、通用响应类 - BaseResponse

```java
@Data
public class BaseResponse<T> implements Serializable {

    private int code;
    private T data;
    private String message;

    public BaseResponse(int code, T data, String message) {
        this.code = code;
        this.data = data;
        this.message = message;
    }

    public BaseResponse(int code, T data) {
        this(code, data, "");
    }

    public BaseResponse(ErrorCode errorCode) {
        this(errorCode.getCode(), null, errorCode.getMessage());
    }
}
```

#### 5、响应工具类 - ResultUtils

```java
public class ResultUtils {

    public static <T> BaseResponse<T> success(T data) {
        return new BaseResponse<>(0, data, "ok");
    }

    public static BaseResponse<?> error(ErrorCode errorCode) {
        return new BaseResponse<>(errorCode);
    }

    public static BaseResponse<?> error(int code, String message) {
        return new BaseResponse<>(code, null, message);
    }

    public static BaseResponse<?> error(ErrorCode errorCode, String message) {
        return new BaseResponse<>(errorCode.getCode(), null, message);
    }
}
```

#### 6、全局异常处理器 - GlobalExceptionHandler

```java
@RestControllerAdvice
@Slf4j
public class GlobalExceptionHandler {

    @ExceptionHandler(BusinessException.class)
    public BaseResponse<?> businessExceptionHandler(BusinessException e) {
        log.error("BusinessException", e);
        return ResultUtils.error(e.getCode(), e.getMessage());
    }

    @ExceptionHandler(RuntimeException.class)
    public BaseResponse<?> runtimeExceptionHandler(RuntimeException e) {
        log.error("RuntimeException", e);
        return ResultUtils.error(ErrorCode.SYSTEM_ERROR, "系统错误");
    }
}
```

#### 7、分页请求包装 - PageRequest

```java
@Data
public class PageRequest {

    /** 当前页号 */
    private int current = 1;

    /** 页面大小 */
    private int pageSize = 10;

    /** 排序字段 */
    private String sortField;

    /** 排序顺序（默认降序） */
    private String sortOrder = "descend";
}
```

#### 8、删除请求包装 - DeleteRequest

```java
@Data
public class DeleteRequest implements Serializable {

    private Long id;
    private static final long serialVersionUID = 1L;
}
```

#### 9、全局跨域配置 - CorsConfig

```java
@Configuration
public class CorsConfig implements WebMvcConfigurer {

    @Override
    public void addCorsMappings(CorsRegistry registry) {
        registry.addMapping("/**")
                .allowCredentials(true)
                .allowedOriginPatterns("*")
                .allowedMethods("GET", "POST", "PUT", "DELETE", "OPTIONS")
                .allowedHeaders("*")
                .exposedHeaders("*");
    }
}
```

> **注意**：必须用 `allowedOriginPatterns` 而非 `allowedOrigins`，因为 `*` 和 `allowCredentials(true)` 不能同时使用。

### 编写示例接口

## 二、前端项目初始化

| 依赖 | 版本要求 | 说明 |

使用 Vue 官方脚手架 create-vue 创建项目：


### 路由

```typescript
import { createRouter, createWebHistory } from 'vue-router'
import BasicLayout from '@/layouts/BasicLayout.vue'

const router = createRouter({
```

### 请求

#### 1、安装请求工具库

```shell
npm install axios --save
```

#### 2、全局自定义请求（request.ts）

```typescript
import axios from 'axios'
import { message } from 'ant-design-vue'

const myAxios = axios.create({
  baseURL: 'http://localhost:8123',
  timeout: 10000,
  withCredentials: true,
})

// 请求拦截器
myAxios.interceptors.request.use(
  (config) => config,
  (error) => Promise.reject(error)
)

// 响应拦截器
myAxios.interceptors.response.use(
  (response) => {
    const { data } = response
    // 未登录则跳转到登录页
    if (data.code === 40100) {
      message.error('请先登录')
      window.location.href = '/user/login'
    }
    return data
  },
  (error) => {
    message.error('请求失败')
    return Promise.reject(error)
  }
)

export default myAxios
```

#### 3、自动生成请求代码（OpenAPI）

使用 OpenAPI 工具从后端 Knife4j 接口文档自动生成前端请求代码：

```shell
npm install openapi-typescript-codegen --save-dev
```

```typescript
const testApi = async () => {
```

### 全局状态管理

#### 1、引入 Pinia

脚手架默认已安装 Pinia，无需额外操作。

#### 2、定义状态（stores/useLoginUserStore.ts）

```typescript
import { defineStore } from 'pinia'
import { ref } from 'vue'

export const useLoginUserStore = defineStore('loginUser', () => {
  const loginUser = ref<any>(null)

  function setLoginUser(user: any) {
    loginUser.value = user
  }

  function clearLoginUser() {
    loginUser.value = null
  }

  return { loginUser, setLoginUser, clearLoginUser }
})
```

#### 3、使用状态

```vue
<script setup lang="ts">
import { useLoginUserStore } from '@/stores/useLoginUserStore'
const loginUserStore = useLoginUserStore()
</script>

<template>
  <div v-if="loginUserStore.loginUser">
    欢迎，{{ loginUserStore.loginUser.userName }}
  </div>
</template>
```

#### 4、测试全局状态管理

确保在多个组件间状态共享正常。

### 页面开发流程

标准的前端页面开发流程：

1. **新建页面文件**：`views/xxxPage.vue`
2. **配置路由**：`router/index.ts` 中添加路由映射
3. **编写组件**：模板 + 逻辑 + 样式
4. **对接接口**：使用 `myAxios` 调用后端 API
5. **联调测试**：确保前后端数据交互正常

## 关键注意事项

| 注意点 | 说明 |
|--------|------|
| **JDK 版本 ≤ 17** | 后续 Caffeine 缓存库要求 JDK 11+ |
| **Server URL** | 必须改为 `https://start.aliyun.com/` |
| **跨域配置** | 用 `allowedOriginPatterns("*")` 而非 `allowedOrigins("*")` |
| **Axios withCredentials** | `true` 确保携带 Cookie/Session |
| **前端代理** | 开发环境建议配置 Vite 代理避免跨域 |
| **ESLint 类型检查** | 构建时注意类型规范，否则打包失败 |
