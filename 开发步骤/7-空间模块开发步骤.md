# 空间模块 - 开发步骤指南

本文档基于"智能协同云图库项目教程"第7节整理，涵盖空间管理、用户创建私有空间、权限控制、空间级别与限额控制的完整开发步骤。

---

## 一、需求分析

1. **【管理员】管理空间**：可对整个系统中的空间进行增删改查管理
2. **用户创建私有空间**：用户可创建最多一个私有空间，在空间内自由管理图片
3. **私有空间权限控制**：用户仅能访问和管理自己的私有空间，私有空间的图片不展示在公共图库，也不需要管理员审核
4. **空间级别和限额控制**：每个空间有不同级别（普通版/专业版/旗舰版），对应不同的容量和图片数量限制，超出限制无法继续上传

> 引入"空间"概念的原因：私有图片需要隐私，与公共图库逻辑分离。抽象空间概念便于后续扩展团队共享空间。

## 二、方案设计

### 1. 空间库表设计

#### space 表

```sql
-- 空间表
create table if not exists space
(
    id         bigint auto_increment comment 'id' primary key,
    spaceName  varchar(128)                       null comment '空间名称',
    spaceLevel int      default 0                 null comment '空间级别：0-普通版 1-专业版 2-旗舰版',
    maxSize    bigint   default 0                 null comment '空间图片的最大总大小',
    maxCount   bigint   default 0                 null comment '空间图片的最大数量',
    totalSize  bigint   default 0                 null comment '当前空间下图片的总大小',
    totalCount bigint   default 0                 null comment '当前空间下的图片数量',
    userId     bigint                             not null comment '创建用户 id',
    createTime datetime default CURRENT_TIMESTAMP not null comment '创建时间',
    editTime   datetime default CURRENT_TIMESTAMP not null comment '编辑时间',
    updateTime datetime default CURRENT_TIMESTAMP not null on update CURRENT_TIMESTAMP comment '更新时间',
    isDelete   tinyint  default 0                 not null comment '是否删除',
    -- 索引设计
    index idx_userId (userId),
    index idx_spaceName (spaceName),
    index idx_spaceLevel (spaceLevel)
) comment '空间' collate = utf8mb4_unicode_ci;
```

> 额外操作：在数据库中执行上述 DDL 创建 space 表。

#### picture 表新增字段

```sql
-- 添加新列
ALTER TABLE picture
    ADD COLUMN spaceId  bigint  null comment '空间 id（为空表示公共空间）';

-- 创建索引
CREATE INDEX idx_spaceId ON picture (spaceId);
```

> spaceId 默认为 null，表示图片上传到了公共图库。

### 2. 公共图库和空间的关系

- 公共图库与私有空间独立设计，**不混合**
- 公共图库：无需登录，无额度限制，上传到 `public/` 目录
- 私有空间：需登录，需权限控制，有额度限制，上传到 `space/` 目录

```java
@Override
public void validSpace(Space space, boolean add) {
    ThrowUtils.throwIf(space == null, ErrorCode.PARAMS_ERROR);
    String spaceName = space.getSpaceName();
    Integer spaceLevel = space.getSpaceLevel();
    SpaceLevelEnum spaceLevelEnum = SpaceLevelEnum.getEnumByValue(spaceLevel);

    if (add) {

        if (spaceLevel == null) {

    if (spaceLevel != null && spaceLevelEnum == null) {
        throw new BusinessException(ErrorCode.PARAMS_ERROR, "空间级别不存在");
    }
    if (StrUtil.isNotBlank(spaceName) && spaceName.length() > 30) {
        throw new BusinessException(ErrorCode.PARAMS_ERROR, "空间名称过长");
    }
}
```

### 模块二：创建私有空间

#### 步骤1：方案设计

创建空间的流程：
1. 填充参数默认值（空间名默认"默认空间"，级别默认普通版）
2. 校验参数
3. 校验权限：非管理员只能创建普通级别的空间
4. 控制同一用户只能创建一个私有空间（**本地锁 + 事务**）

#### 步骤2：创建空间服务

```java
@Resource
private TransactionTemplate transactionTemplate;

@Override
public long addSpace(SpaceAddRequest spaceAddRequest, User loginUser) {
    // 实体类和 DTO 转换
    Space space = new Space();
    BeanUtils.copyProperties(spaceAddRequest, space);
    // 默认值
    if (StrUtil.isBlank(spaceAddRequest.getSpaceName())) {
        space.setSpaceName("默认空间");
    }
    if (spaceAddRequest.getSpaceLevel() == null) {
        space.setSpaceLevel(SpaceLevelEnum.COMMON.getValue());
    }
    // 填充数据
    this.fillSpaceBySpaceLevel(space);
    // 数据校验
    this.validSpace(space, true);
    Long userId = loginUser.getId();
    space.setUserId(userId);
    // 权限校验
    if (SpaceLevelEnum.COMMON.getValue() != spaceAddRequest.getSpaceLevel()
            && !userService.isAdmin(loginUser)) {
        throw new BusinessException(ErrorCode.NO_AUTH_ERROR, "无权限创建指定级别的空间");
    }
    // 针对用户加锁
    String lock = String.valueOf(userId).intern();
    synchronized (lock) {
        Long newSpaceId = transactionTemplate.execute(status -> {
            boolean exists = this.lambdaQuery()
                    .eq(Space::getUserId, userId).exists();
            ThrowUtils.throwIf(exists, ErrorCode.OPERATION_ERROR, "每个用户仅能有一个私有空间");
            // 写入数据库
            boolean result = this.save(space);
            ThrowUtils.throwIf(!result, ErrorCode.OPERATION_ERROR);
            return space.getId();
        });
        return Optional.ofNullable(newSpaceId).orElse(-1L);
    }
}
```

> 核心：使用 `synchronized` 对 `userId` 加锁 + `transactionTemplate` 编程式事务。不同用户获取不同的锁，性能影响低。

#### 扩展：本地锁优化

使用 `ConcurrentHashMap` 存储锁对象，防止内存泄漏：

```java
Map<Long, Object> lockMap = new ConcurrentHashMap<>();

public long addSpace(SpaceAddRequest spaceAddRequest, User user) {
    Long userId = user.getId();
    Object lock = lockMap.computeIfAbsent(userId, key -> new Object());
    synchronized (lock) {
        try {
            // 数据库操作
        } finally {
            lockMap.remove(userId);  // 防止内存泄漏
        }
    }
}
```

#### 步骤3：封装统一的图片权限校验方法

```java
@Override
public void checkPictureAuth(User loginUser, Picture picture) {
    Long spaceId = picture.getSpaceId();
    if (spaceId == null) {
        // 公共图库，仅本人或管理员可操作
        if (!picture.getUserId().equals(loginUser.getId()) && !userService.isAdmin(loginUser)) {
            throw new BusinessException(ErrorCode.NO_AUTH_ERROR);
        }
    } else {
        // 私有空间，仅空间管理员可操作
        if (!picture.getUserId().equals(loginUser.getId())) {
            throw new BusinessException(ErrorCode.NO_AUTH_ERROR);
        }
    }
}
```

#### 步骤4：删除图片封装为 Service 方法

```java
@Override
public void deletePicture(long pictureId, User loginUser) {
    ThrowUtils.throwIf(pictureId <= 0, ErrorCode.PARAMS_ERROR);
    ThrowUtils.throwIf(loginUser == null, ErrorCode.NO_AUTH_ERROR);
    // 判断是否存在
    Picture oldPicture = this.getById(pictureId);
    ThrowUtils.throwIf(oldPicture == null, ErrorCode.NOT_FOUND_ERROR);
    // 校验权限（调用统一的权限校验方法）
    checkPictureAuth(loginUser, oldPicture);
    // 操作数据库（在事务中完成，同时更新空间额度）
    transactionTemplate.execute(status -> {
        boolean result = this.removeById(pictureId);
        ThrowUtils.throwIf(!result, ErrorCode.OPERATION_ERROR);
        // 释放额度
        Long spaceId = oldPicture.getSpaceId();
        if (spaceId != null) {
            boolean update = spaceService.lambdaUpdate()
                    .eq(Space::getId, spaceId)
                    .setSql("totalSize = totalSize - " + oldPicture.getPicSize())
                    .setSql("totalCount = totalCount - 1")
                    .update();
            ThrowUtils.throwIf(!update, ErrorCode.OPERATION_ERROR, "额度更新失败");
        }
        return true;
    });
    // 异步清理文件
    this.clearPictureFile(oldPicture);
}
```

## 关键技术点

| 技术点 | 说明 |
|--------|------|
| **空间抽象** | 将私有空间与公共图库完全分离，减少对原有代码的修改，便于后续扩展团队空间 |
| **本地锁 + 事务** | 使用 `synchronized` 对 userId 加锁 + `transactionTemplate` 编程式事务，保证同一用户只能创建一个私有空间 |
| **权限分级** | 公共图库（本人或管理员可操作）vs 私有空间（仅空间管理员可操作），通过 `checkPictureAuth` 统一管理 |
| **额度同步** | 上传和删除图片时在事务中同步更新 `totalSize` 和 `totalCount`，避免数据不一致 |
| **按空间划分目录** | 公共图库上传到 `public/`，私有空间上传到 `space/{spaceId}/` |
| **前端路由重定向** | 使用 `router.replace` 从中间页跳转，避免用户按后退按钮回到中间页 |
